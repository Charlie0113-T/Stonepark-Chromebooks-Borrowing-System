

## 🚀 **Vercel 部署提示词**

### **前置检查清单**

- [ ] Backend 和 Frontend 都已在本地成功运行
- [ ] 确认所有环境变量已识别
- [ ] 数据库策略已决定（云 PG / SQLite + 同步）

---

### **部署策略**

**推荐方案：前后端分离部署**
- **前端**：部署到 Vercel（React SPA）→ `https://chromebook-app.vercel.app`
- **后端**：部署到 Vercel Serverless Functions / 独立 Node.js 服务（Railway / Render / Heroku）
  - 原因：SQLite 无法持久化在 Vercel 临时文件系统

**或者（简化版）：Monorepo 部署**
- 整个项目部署到 Vercel，后端使用 `/api/*` Serverless Functions
- 需要迁移至 PostgreSQL 或 MongoDB（持久化）

### **生成的 Vercel 部署提示词**

将这段内容保存为 **`.vercel-deployment.md`** 或放入项目 README：

```markdown
# Vercel 部署指南 - Stonepark Chromebook Manager

## 📦 前期准备

### 1. 环境变量配置
在所有部署环境中需要设置以下变量：

**生产环境（`.env.production`）：**
```bash
# Backend
JWT_SECRET=your-super-secret-key-min-32-chars-long
PORT=3001
NODE_ENV=production
DATABASE_URL=postgresql://user:pass@host/dbname  # 或使用下面的 SQLite 远程同步
AUTH_BYPASS=false

# Google OAuth（从 Google Cloud Console 获取）
GOOGLE_CLIENT_ID=your-client-id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=your-client-secret
GOOGLE_CALLBACK_URL=https://your-api-domain.com/api/auth/google/callback

# Frontend
REACT_APP_API_URL=https://your-api-domain.com
REACT_APP_REQUIRE_AUTH=true
```

**开发环境（`.env.development`）：**
```bash
REACT_APP_API_URL=http://localhost:4000
REACT_APP_REQUIRE_AUTH=false
```

### 2. 为 Vercel 创建 `.env.example`
```bash
# backend/.env.example
JWT_SECRET=your-secret-here
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret
GOOGLE_CALLBACK_URL=http://localhost:4000/api/auth/google/callback
AUTH_BYPASS=false
PORT=4000

# frontend/.env.example
REACT_APP_API_URL=http://localhost:4000
REACT_APP_REQUIRE_AUTH=false
```

### 3. 创建 `vercel.json` 配置

**根目录 vercel.json（整体部署）：**
```json
{
  "version": 2,
  "buildCommand": "npm install --prefix backend && npm install --prefix frontend && npm run build --prefix frontend",
  "outputDirectory": "frontend/build",
  "env": {
    "JWT_SECRET": "@jwt_secret",
    "GOOGLE_CLIENT_ID": "@google_client_id",
    "GOOGLE_CLIENT_SECRET": "@google_client_secret"
  },
  "functions": {
    "backend/api/**/*.js": {
      "memory": 1024,
      "maxDuration": 60
    }
  }
}
```

**后端 Serverless 路由（`backend/vercel.json`）：**
```json
{
  "version": 2,
  "buildCommand": "npm install",
  "functions": {
    "index.js": {
      "memory": 1024,
      "maxDuration": 60,
      "runtime": "nodejs20.x"
    }
  },
  "routes": [
    {
      "src": "/api/(.*)",
      "dest": "/index.js"
    }
  ]
}
```

## 🗄️ 数据库部署方案

### 方案 A：迁移到 PostgreSQL（推荐）

1. **创建云数据库**
   - Supabase（https://supabase.com）- 推荐，免费层足够
   - Railway（https://railway.app）
   - Render（https://render.com）

2. **环境变量**
   ```bash
   DATABASE_URL=postgresql://user:password@host:5432/dbname
   ```

3. **更新后端数据库适配器**
   ```javascript
   // backend/src/db/database.js 改为 pg 驱动
   // 考虑使用 ORM（Sequelize / Prisma）简化迁移
   ```

### 方案 B：SQLite + 文件远程同步

1. 使用 Vercel 的持久化存储（如 Supabase Storage）
2. 启动时下载，关闭时上传
3. **不推荐**：性能差，并发冲突

## 📋 Vercel 部署步骤

### 1. 连接 GitHub

```bash
# 第一次部署
vercel login
vercel
```

### 2. 配置项目设置

在 Vercel 项目设置中：
- **Root Directory**：`.` （根目录）
- **Build Command**：`npm run build:frontend` 或自定义
- **Output Directory**：`frontend/build`
- **Environment Variables**：添加所有 `.env.production` 变量

### 3. Serverless Functions 部署（后端）

创建 `api/index.js` 入口（Vercel 自动识别 `/api/*` 为 Serverless）：

```javascript
// api/index.js
const app = require('../backend/index');

module.exports = (req, res) => {
  return app(req, res);
};
```

或直接配置：
```json
{
  "rewrites": [
    { "source": "/api/(.*)", "destination": "/backend/index.js" }
  ]
}
```

### 4. 部署

```bash
vercel --prod
```

## 🐛 常见问题

### 问题 1：CORS 错误
**原因**：前端域名未在后端 CORS 白名单中
**解决**：
```javascript
// 添加到 backend/index.js
app.use(cors({
  origin: process.env.CORS_ORIGIN?.split(',') || ['*'],
  credentials: true
}));
```

### 问题 2：SQLite 数据丢失
**原因**：Vercel 无持久化文件系统
**解决**：迁移到 PostgreSQL（见上文）

### 问题 3：超时错误
**出现于**：Serverless 函数执行时间过长（数据库操作慢）
**解决**：
- 增加 `maxDuration`（最长 60s）
- 优化数据库查询
- 使用连接池

### 问题 4：Google OAuth 失败
**检查清单**：
1. `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` 已设置
2. `GOOGLE_CALLBACK_URL` 与 Google Console 中注册的一致
3. 例：`https://api.yourdomain.com/api/auth/google/callback`
